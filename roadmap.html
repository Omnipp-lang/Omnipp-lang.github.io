Omni++ Public Roadmap

Omni++ is a next-generation programming language — the spiritual successor to C++ — combining low-level control with high-level capabilities, Web3 support via Crypted, and full interoperability with major programming ecosystems.

Phase 1: Core Language & Compiler Foundations (Q2 2025)

  • Design core Omni++ syntax
  • Integrate AOT/JIT switchable compilation
  • LLVM backend integration
  • Custom closed-source backend
  • Core standard library (IO, memory, math, string, threading)
  • REPL & scripting support
  • Initial documentation & syntax guide

Phase 2: Superset & Interop Expansion (Q3 2025)

  • FFI for C, C++, C#, JavaScript, and Python
  • Built-in Web3 tooling via Crypted
  • ML/AI: TensorFlow, PyTorch C++, ONNX Runtime
  • Parallelism: Checked C, Rust interop, RxCpp
  • Functional: SQL++, Lua++, Jupyter kernel
  • GUI: Qt, wxWidgets, JUCE, ImGui, Vulkan-HPP
  • Embedded: Zephyr, TinyOS, MISRA, AUTOSAR
  • Game engines: Godot, Unreal, CryEngine, Unity
  • Scientific: Verilog/VHDL transpiler, Geometric Algebra DSL
  • WASM-native, Zig/Nim/D language bridges

Phase 3: Developer Experience & Tooling (Q4 2025)

  • CLI Toolchain (omnic, omnijit, omnipkg)
  • VSCode extension & syntax highlighting
  • Omni++ Language Server (LSP)
  • Unit testing framework
  • Package manager (omniverse)
  • Interactive documentation & example portal
  • Jupyter integration for AI/data science workflows

Phase 4: Public Beta & Ecosystem (Q1 2026)

  • Closed Alpha with selected contributors
  • Public Beta for Linux, Windows, macOS
  • Release on GitHub & PyPI
  • Launch public issue tracker & forums
  • Start ecosystem grant program
  • Launch official Discord/Matrix community

Phase 5: Omni++ 1.0 Stable (Mid–Late 2026)

  • Optimization and final performance tuning
  • Security audit & fuzz testing
  • Long-term support (LTS) version
  • Omni++ 1.0 Stable Release
  • Official showcase of Omni++ projects
